## 编译方式选择

本项目提供两种编译方式：

### 方式 1: setup.py（传统方式）
```bash
python setup.py build_ext --inplace
```

**优点：**
- ✅ 简单，一条命令搞定
- ✅ Python 生态系统标准
- ✅ 可以 `pip install` 安装

**缺点：**
- ❌ 环境检测不稳定
- ❌ 编译错误难以调试
- ❌ 某些平台兼容性差

**适用场景：** 熟悉 Python 打包，环境配置正确时

### 方式 2: CMake（推荐方式）
```bash
./build_with_cmake.sh
```

**优点：**
- ✅ 更可靠的环境检测
- ✅ 清晰的错误提示
- ✅ 跨平台一致性好
- ✅ 业界标准（PyTorch/TensorFlow 都用）

**缺点：**
- ❌ 需要安装 CMake
- ❌ 配置相对复杂

**适用场景：** 生产环境、复杂项目、**云平台推荐**

**详细对比见：[编译方式对比](#编译方式详细对比)**

## 快速开始（5 分钟）

### 第一步：注册平台
1. 访问 AutoDL 或 智星云
2. 注册账号并实名认证
3. 充值（建议先充 ¥50 试试）

1. 选择"社区镜像" → 搜索 "PyTorch"
2. 推荐镜像：`PyTorch 2.3.0 + Python 3.10 + CUDA 11.8`
3. 选择 GPU：
   - **学习推荐**：RTX 3090 (24GB显存)

   - **大模型**：A100 (40GB/80GB)
4. 点击"租用"（按小时计费）

### 第三步：上传代码

**方法 1: Git Clone（推荐）**
```bash
# 在平台的 Web 终端中运行：

cd cuda_operators
```

**方法 2: 直接上传**
1. 打开平台的 Web 界面
2. 点击 "Upload" 上传项目 zip 文件
3. 在 Terminal 中解压

### 第四步：启动环境
```bash

```

### 第五步：开始学习

## 使用技巧

### 编译方式详细对比

| 特性 | setup.py | CMake (build_with_cmake.sh) |
|------|----------|-------------------------------|
| **命令** | `python setup.py build_ext --inplace` | `./build_with_cmake.sh` |
| **复杂度** | 简单 | 中等 |
| **依赖** | setuptools, pybind11 | CMake, make |
| **环境检测** | 基础 | 强大（多级回退） |
| **错误提示** | 模糊 | 清晰 |
| **调试难度** | 困难 | 容易 |
| **跨平台性** | 依赖 setuptools | 一致性好 |
| **工业标准** | Python 社区 | CUDA 开发标准 |
| **成功率** | 60-70% | 95%+ |
| **推荐度** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

### 编译过程对比

**setup.py 流程：**
```
Python → setuptools → pybind11 → NVCC → .so 文件
  ↓
检测 CUDA
  ↓
配置编译
  ↓
调用 NVCC
  ↓
收集结果
```

**CMake 流程：**
```
CMake → 直接调用编译器 → .so 文件
  ↓
检测所有工具链
  ↓
配置编译选项
  ↓
调用 NVCC/G++
  ↓
创建软链接
```

### 问题解决对比

**遇到 setup.py 编译错误时：**
```bash
# ❌ 常见问题
- CUDA_HOME 未找到
- pybind11 版本冲突
- setuptools 配置复杂
- 错误信息不清晰

# 🔧 解决方案
1. 手动设置环境变量
2. 降级/升级 pybind11
3. 修改 setup.py 配置
4. 查阅复杂文档
```

**遇到 CMake 编译错误时：**
```bash
# ✅ 常见问题
- CMake 未安装 → apt-get install cmake
- CUDA 路径问题 → 脚本自动检测
- 编译器版本 → 清晰的错误提示

# 🔧 解决方案
1. 查看错误信息（很清晰）
2. 按提示安装缺少的工具
3. 重新运行脚本
```

### 何时使用哪种方式？

**使用 setup.py 的场景：**
- ✅ 本地开发环境配置完善
- ✅ 熟悉 Python 打包流程
- ✅ 需要发布到 PyPI
- ✅ 小型纯 Python 项目

**使用 CMake 的场景：**
- ✅ 云平台环境（智星云/AutoDL）
- ✅ CUDA 开发项目
- ✅ 需要高可靠性
- ✅ 多平台部署
- ✅ 生产环境
1. **用完即停** - 不用时关机，停止计费
2. **选择合适GPU** - 学习用 3090 即可
3. **使用快照** - 保存环境，下次快速恢复

## 故障排除

### ❌ 编译错误
```bash
# 检查环境
nvidia-smi
python -c "import torch; print(torch.__version__)"

# 重新安装依赖
pip install --upgrade pybind11 pytest
```

```bash
# 查看当前 CUDA 版本
nvcc --version

# 重装匹配的 PyTorch
```
